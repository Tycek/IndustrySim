# Market Stockpile Redesign — Implementation Plan

## Goal

Replace the current random buy/sell offer generation and drift-based price adjustment with a
stockpile-driven market. The market represents external world demand. It holds a stockpile per
resource, posts one persistent buy offer per resource (capacity-limited), consumes its stockpile
at a fixed rate each turn, and derives prices directly from the stockpile fill ratio.

No sell offers are generated by the market. Raw materials only enter the economy through player
and AI company industries (mines).

---

## New Market State

Add to `Market.cs`:

```csharp
// Current stockpile level per resource (units held by the market).
public Dictionary<string, double> StockpileLevel { get; set; }

// Maximum stockpile capacity per resource. Grows slowly each turn.
public Dictionary<string, double> StockpileCapacity { get; set; }

// Units the market consumes from its stockpile per turn (fixed baseline).
public static readonly IReadOnlyDictionary<string, double> ConsumptionRates
```

### Starting Capacity (baseline values, tuned to feel right)

Rough starting capacities should be proportional to how fast AI/player industries can produce.
A capacity of ~10× the consumption rate gives ~10 turns of buffer at zero production.

Example consumption rates (units/turn):
- Coal: 15, Iron Ore: 10, Coal Coke: 8, Steel Ingot: 5
- Steel Wire/Bar/Plate: 4 each
- Copper Ore: 8, Copper Ingot: 4, Copper Wire/Pipe: 3 each
- Crude Oil: 12, Gas/Diesel/Kerosene: 5 each
- Chemicals: 4, Plastic Resin: 4, Plastics: 5, Fertiliser: 3, Synthetic Rubber: 3

Starting capacities = ConsumptionRate × 10 (adjust during playtesting).

### Game-Start Randomization

At `GameLoop.StartNew`, randomize the initial `StockpileLevel` for each resource:

```
StockpileLevel[r] = StockpileCapacity[r] × rng.NextDouble() in [0.20, 0.80]
```

This ensures no two games start with the same price landscape.

---

## Price Formula

Remove `AdjustPrices(sellPressure, buyPressure)` entirely.

Price is now a pure function of fill ratio, computed at the start of each turn (or on demand):

```
fillRatio = StockpileLevel[r] / StockpileCapacity[r]   // clamped to [0, 1]
PriceIndex[r] = BasePrices[r] × Lerp(PriceCeiling, PriceFloor, fillRatio)
```

With the existing constants `PriceFloor = 0.40` and `PriceCeiling = 5.00`:
- Empty stockpile  → 5.00× base price
- Half-full        → 2.7× base price  (midpoint of 5.00 and 0.40)
- Full stockpile   → 0.40× base price

`PreviousPriceIndex` is still updated each turn for trend arrows in the UI.

---

## Offer Model

### One Persistent Buy Offer Per Resource

Replace the randomized multi-offer generation with one persistent `MarketOffer` per resource:

- `Type = OfferType.Buy`
- `Source = "Market"`
- `TurnsRemaining = int.MaxValue` (never expires naturally)
- `Quantity` = how much market *still wants* this turn = `StockpileCapacity[r] - StockpileLevel[r]`
  clamped to a sane max per turn (e.g. `ConsumptionRate[r] × 3`) so players cannot dump
  unlimited stock in one turn
- `PricePerUnit = PriceIndex[r]` (recomputed each turn before offers are refreshed)

If `StockpileLevel[r] >= StockpileCapacity[r]`, no offer is posted for that resource (stockpile full).

Offer quantities should be rounded to the nearest 5 for readability.

### No Market Sell Offers

`GenerateOffers` stops generating any `OfferType.Sell` offers from `Source = "Market"`.
Sell offers may still be posted by AI companies and (future) by the player.

### Accepting a Buy Offer

When player or AI accepts a buy offer (sells to the market):
- Deduct resource from seller inventory
- Credit seller balance
- **Increase `StockpileLevel[r]`** by the quantity sold
- Recompute price and update the offer quantity for that resource

---

## Turn Tick Sequence (replaces current GenerateOffers logic)

Each turn in `Market.TickStockpiles()`:

1. **Consume**: `StockpileLevel[r] -= ConsumptionRates[r]`, floor at 0.
2. **Grow capacity**: `StockpileCapacity[r] *= (1 + GrowthRatePerTurn)`.
   Suggested `GrowthRatePerTurn = 0.002` (0.2% per turn ≈ ~11% per 50 turns).
3. **Recompute prices** from fill ratio.
4. **Refresh offers**: rebuild the one buy offer per resource based on new stockpile state.
5. **Tick random events**: apply active event modifiers to consumption or stockpile level,
   decrement remaining turns, remove expired events.

`GameLoop.ProcessTurn` calls `market.TickStockpiles()` instead of the current
`market.GenerateOffers(rng)` + `market.AdjustPrices(...)` pair.

---

## Random Events

Add to `Market.cs`:

```csharp
public List<MarketEvent> ActiveEvents { get; set; } = [];
```

```csharp
public record MarketEvent(
    string ResourceName,
    double StockpileShiftPerTurn,  // negative = drains faster, positive = fills faster
    int TurnsRemaining,
    string Description
);
```

Events are applied in step 5 of `TickStockpiles` — `StockpileLevel[r]` is shifted by
`StockpileShiftPerTurn` each turn (clamped to capacity bounds).

### Event Generation

`GameLoop.ProcessTurn` rolls for a new event with low probability (e.g. 5% per turn).
Events should feel rare but impactful. Example events:

| Description                   | Resource       | ShiftPerTurn | Duration   |
|-------------------------------|----------------|--------------|------------|
| Construction boom             | Steel Ingot    | -8           | 8–12 turns |
| Infrastructure drive          | Steel Plate    | -6           | 6–10 turns |
| Oil glut                      | Crude Oil      | +15          | 5–8 turns  |
| Electrical grid expansion     | Copper Wire    | -5           | 8–15 turns |
| Plastics surplus              | Plastics       | +10          | 4–7 turns  |
| Chemical plant shutdown       | Chemicals      | -6           | 5–10 turns |
| Mining strike                 | Coal           | -10          | 3–6 turns  |

Events are reported in `TurnEvents` and shown in the UI notification banner.

## Changes by File

### `Market.cs`
- Add `StockpileLevel`, `StockpileCapacity`, `ConsumptionRates`, `ActiveEvents`.
- Add `TickStockpiles(Random rng)` — replaces `GenerateOffers` + `AdjustPrices`.
- Add `ComputePrice(string resource)` — lerp formula.
- Remove `AdjustPrices(sellPressure, buyPressure)`.
- Modify `GenerateOffers` → remove or repurpose; all offer logic moves to `TickStockpiles`.
- When a buy offer is accepted (resource sold to market), `StockpileLevel` must be updated.
  This means offer acceptance in `GameLoop`, `Player`, and `AiCompany` needs to call a market method
  rather than just removing the offer. Consider `market.AcceptBuyOffer(resourceName, quantity)`.

### `GameLoop.cs`
- Replace `market.GenerateOffers(rng)` + `market.AdjustPrices(...)` calls with `market.TickStockpiles(rng)`.
- Remove `sellPressure` / `buyPressure` accumulation logic.
- Add random event generation (roll + pick from event table, add to `market.ActiveEvents`).
- Propagate active event names into `TurnEvents` for UI display.
- Pass random stockpile initialization into `GameLoop.StartNew`.

### `AiCompany.cs`
- `AcceptOffers`: when accepting a market buy offer, call `market.AcceptBuyOffer(...)` to update stockpile.
- `PostToMarket`: no change needed for posting; AI still posts its own sell/buy offers as before.

### `Player.cs` / `GameLoop.TryAcceptOffer`
- Same stockpile update on market buy offer acceptance.

### `TurnEvents.cs`
- Add `IReadOnlyList<string> MarketEvents` (event descriptions that fired this turn).

### `Market.cs` (constructor / `StartNew`)
- Initialize `StockpileCapacity` from baseline table.
- Initialize `StockpileLevel` with randomized fill ratio `[0.20, 0.80]` × capacity.

## AI Company Integration — Expected Margin Formula

Once prices are driven by stockpile fill ratios, the AI building decision improves significantly.

In `AiCompany.ConsiderBuildingIndustry`, add a margin filter before the weighted pick:

```csharp
private static decimal ComputeExpectedMargin(IIndustry industry, IReadOnlyDictionary<string, decimal> priceIndex)
{
    var revenue = industry.OutputsProduced.Sum(o =>
        (decimal)o.Quantity * priceIndex.GetValueOrDefault(o.Name, 0m));
    var inputCost = industry.InputsRequired.Sum(i =>
        (decimal)i.Quantity * priceIndex.GetValueOrDefault(i.Name, 0m));
    return revenue - inputCost - industry.RunningCost;
}
```

Filter `candidates` to only those with `margin > 0` before `WeightedPick`. If no candidates
pass, skip building entirely — no point building an unprofitable industry.

`AiCompanyGenerator.Score()` stays as-is (supply/demand balance for *which* industry to build);
the margin formula acts as a gate (*whether* to build at all).

## What Does NOT Change

- `Contract` model and bilateral contract settlement — unchanged.
- AI company `AcceptContracts` / `PostToMarket` surplus/deficit logic — unchanged.
- `SimulationRunner` — uses fixed prices, no market interaction; unchanged.
- `IMarketParticipant` interface — unchanged.
- All industry classes — unchanged.
